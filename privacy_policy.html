TRERO

プライバシーポリシー

1. 本ポリシーについて

本ポリシーは、Treroがユーザー情報をどのように取得、利用、保存、削除するかを説明するものです。利用前にご確認ください。

2. 取得する情報

区分 主な項目 利用目的
アカウント情報 UID、メールアドレス、表示名、認証プロバイダ情報、作成日時 ログイン、アカウント管理、公開プロフィール同期
ワークアウト記録 種目ID、種目名、部位、日付、セット番号、重量、回数、メモ、記録時刻 履歴表示、分析、目標進捗計算、CSV出力機能の提供
体組成データ 体重、体脂肪率、筋肉率、ウエスト、メモ、記録日時 体組成履歴、グラフ、BMI等の派生値計算
ソーシャル関連 公開プロフィール、フレンド申請、承認済み関係、ブロック、通報、リアクション フレンド機能とリアクション機能の提供
ローカル設定・端末保存 単位系、身長、目標体重、レストタイマー音、利用状況に応じた案内表示用の完了回数、オンボーディング状態、ローカル保存のプロフィール画像 表示設定、計算、端末内のユーザー体験維持

生体認証用資格情報 メールログイン用の保存済み資格情報（Keychain） Face ID / Touch ID ログイン

取得した情報は、アカウント機能、履歴表示、分析機能、フレンド機能、利用状況計測、サポート対応のために利用されます。保存データと利用状況計測データは、目的を分けて取り扱います。

3. 任意アクセス

  • HealthKit: 体重と体脂肪率の保存時書込、および体組成入力画面での手動取込に使用します。許可しない場合でも基本機能は利用できます。
  • 通知: レストタイマー終了、当日20:00の未記録リマインダー、毎週月曜9:00の週次サマリー通知に使用します。
  • カメラ: フレンド追加用QRコードの読み取りに使用します。
  • Face ID / Touch ID: 保存済み資格情報を使う生体認証ログインに使用します。
  • HealthKit: Used to write weight and body fat percentage when saving, and to manually import the latest values from the body-record screen. Core app functions remain available if permission is not granted.
  • Notifications: Used for rest-timer completion, a same-day 20:00 reminder when no workout has been logged, and a weekly summary every Monday at 09:00.
  • Camera: Used to scan QR codes for adding friends.
  • Face ID / Touch ID: Used for biometric sign-in when saved credentials are available.

5. 保存場所

  • クラウド保存: Firebase / Google Cloud 上にアカウント、ワークアウト、体組成、フレンド関連データを保存します。
  • 利用状況計測: Firebase Analytics を通じて、イベント情報と利用状態がGoogleの基盤上で処理される場合があります。
  • 端末内保存: 設定値、オンボーディング状態、案内表示に使う完了回数、プロフィール画像、ワークアウトキャッシュ、Keychain資格情報を保存します。
  • HealthKit連携データ: Apple Health の権限内で読み書きされます。
  • Cloud storage: Account, workout, body composition, and friend-related data are stored on Firebase / Google Cloud.
  • Usage analytics: Event information and usage-state properties may be processed on Google infrastructure through Firebase Analytics.
  • On-device storage: Settings values, onboarding state, completion counts used for guidance display, profile images, workout caches, and Keychain credentials are stored locally.
  • HealthKit-linked data: Read and written within the scope of Apple Health permissions.

6. データ削除

アプリ内の設定画面からアカウント削除を実行できます。メールアカウントまたはAppleアカウントでは、削除前に再認証が必要になる場合があります。アカウント削除後は、関連するクラウド保存データおよび認証情報の削除が順次行われます。

7. 子どものプライバシー

本アプリは13歳未満を対象としていません。13歳未満の個人情報を意図的に収集することは想定していません。
